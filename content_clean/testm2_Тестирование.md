2. Что делает команда ss -tuln?
- [ ] A) Показывает активные HTTP-запросы  
- [ ] B) Отображает все прослушиваемые TCP и UDP порты  
- [ ] C) Запускает сетевой сканер  
- [ ] D) Настраивает маршрутизацию

<details>
<summary>Показать ответ</summary>

B — ss -tuln:  
- -t — TCP,  
- -u — UDP,  
- -l — listening (слушающие),  
- -n — без разрешения имён (только цифры).
</details>
4. Ты запустил веб-сервер на VM1 (192.168.10.10), но с VM2 не можешь сделать curl. Ping работает. В чём может быть причина?
- [ ] A) Неправильная маска подсети  
- [ ] B) Веб-сервер слушает только на 127.0.0.1  
- [ ] C) Неверный шлюз по умолчанию  
- [ ] D) Проблема с DNS

<details>
<summary>Показать ответ</summary>

B — если сервис привязан к 127.0.0.1, он недоступен извне. Нужно слушать на 0.0.0.0.
</details>
