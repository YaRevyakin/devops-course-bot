2. Что обеспечивает идемпотентность в Ansible?
- [ ] A) Использование Bash-скриптов  
- [ ] B) Модули, которые проверяют текущее состояние перед изменением  
- [ ] C) Ручной запуск playbook’ов  
- [ ] D) Хранение секретов в Vault

<details>
<summary>Показать ответ</summary>

B — Ansible-модули (например, apt, copy, service) сначала проверяют, нужно ли вносить изменения. Если система уже в нужном состоянии — ничего не делается.
</details>
4. Что такое GitOps?
- [ ] A) Использование Git только для хранения кода  
- [ ] B) Подход, при котором желаемое состояние инфраструктуры хранится в Git, а инструменты (например, Argo CD) автоматически синхронизируют кластер с ним  
- [ ] C) Запуск Git-сервера в Kubernetes  
- [ ] D) Автоматическая сборка образов при пуше в ветку

<details>
<summary>Показать ответ</summary>

B — GitOps делает Git единственным источником правды. Argo CD или Flux следят, чтобы реальное состояние совпадало с описанным в репозитории.
</details>
