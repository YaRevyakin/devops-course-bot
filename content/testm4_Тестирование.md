# Тестирование

---
### 1. Какой инструмент используется для управления облачной инфраструктурой через декларативные конфигурации?
- [ ] A) Ansible  
- [ ] B) Jenkins  
- [ ] C) Terraform  
- [ ] D) Docker

<details>
<summary>Показать ответ</summary>

**C** — Terraform специализируется на управлении инфраструктурой (ВМ, сети, БД) в облаке с помощью декларативного кода.
</details>
---
### 2. Что обеспечивает идемпотентность в Ansible?
- [ ] A) Использование Bash-скриптов  
- [ ] B) Модули, которые проверяют текущее состояние перед изменением  
- [ ] C) Ручной запуск playbook’ов  
- [ ] D) Хранение секретов в Vault

<details>
<summary>Показать ответ</summary>

**B** — Ansible-модули (например, `apt`, `copy`, `service`) сначала проверяют, нужно ли вносить изменения. Если система уже в нужном состоянии — ничего не делается.
</details>
---
### 3. Где описывается пайплайн в GitLab CI?
- [ ] A) `Jenkinsfile`  
- [ ] B) `.github/workflows/ci.yml`  
- [ ] C) `.gitlab-ci.yml`  
- [ ] D) `terraform.tf`

<details>
<summary>Показать ответ</summary>

**C** — GitLab CI активируется наличием файла `.gitlab-ci.yml` в корне репозитория.
</details>
---
### 4. Что такое GitOps?
- [ ] A) Использование Git только для хранения кода  
- [ ] B) Подход, при котором желаемое состояние инфраструктуры хранится в Git, а инструменты (например, Argo CD) автоматически синхронизируют кластер с ним  
- [ ] C) Запуск Git-сервера в Kubernetes  
- [ ] D) Автоматическая сборка образов при пуше в ветку

<details>
<summary>Показать ответ</summary>

**B** — GitOps делает Git единственным источником правды. Argo CD или Flux следят, чтобы реальное состояние совпадало с описанным в репозитории.
</details>
---
### 5. Как безопасно хранить пароли в Terraform?
- [ ] A) В файле `main.tf`  
- [ ] B) В комментариях  
- [ ] C) В переменных через `.tfvars`, добавленных в `.gitignore`, или через внешние системы (Vault, SSM)  
- [ ] D) В выводе (`output`)

<details>
<summary>Показать ответ</summary>

**C** — Секреты никогда не должны попадать в Git. Используй `.tfvars` (исключён из версий) или интеграцию с HashiCorp Vault / AWS Secrets Manager.
</details>
