### 1. Какой тип облачного хранилища лучше всего подходит для резервных копий и статических файлов?
- [ ] A) Блочное (Block Storage)  
- [ ] B) Файловое (File Storage)  
- [ ] C) Объектное (Object Storage)  
- [ ] D) Локальный SSD

<details>
<summary>Показать ответ</summary>

**C** — объектное хранилище (например, AWS S3, Yandex Object Storage) оптимизировано для хранения больших объёмов неизменяемых данных с высокой отказоустойчивостью и низкой стоимостью.
</details>

### 2. В каком случае предпочтительнее использовать PostgreSQL вместо MongoDB?
- [ ] A) Нужно хранить JSON-документы без фиксированной схемы  
- [ ] B) Требуются транзакции и гарантии целостности данных  
- [ ] C) Ожидается горизонтальное масштабирование на тысячи узлов  
- [ ] D) Данные пишутся один раз и никогда не обновляются

<details>
<summary>Показать ответ</summary>

**B** — PostgreSQL (реляционная БД) обеспечивает ACID-транзакции и строгую целостность данных, что критично для финансовых систем, учёта и т.п.
</details>

### 3. Какой инструмент используется для асинхронной обработки задач между микросервисами?
- [ ] A) Redis  
- [ ] B) RabbitMQ  
- [ ] C) Prometheus  
- [ ] D) HAProxy

<details>
<summary>Показать ответ</summary>

**B** — RabbitMQ — это брокер сообщений, который позволяет передавать задачи между сервисами через очереди, обеспечивая надёжность и декуплинг.
</details>

### 4. Что такое TTL в контексте Redis?
- [ ] A) Протокол шифрования  
- [ ] B) Время жизни ключа (Time To Live)  
- [ ] C) Тип данных  
- [ ] D) Метод репликации

<details>
<summary>Показать ответ</summary>

**B** — TTL (Time To Live) автоматически удаляет ключ из Redis по истечении заданного времени. Это предотвращает утечки памяти.
</details>

### 5. Почему репликация базы данных не заменяет резервное копирование?
- [ ] A) Репликация слишком медленная  
- [ ] B) Реплика копирует все операции, включая ошибочные (например, DROP TABLE)  
- [ ] C) Реплика хранится в том же дата-центре  
- [ ] D) Реплика не поддерживает чтение

<details>
<summary>Показать ответ</summary>

**B** — репликация синхронизирует **все изменения**, включая человеческие ошибки. Только бэкап позволяет восстановить данные на момент **до** ошибки.
</details>
