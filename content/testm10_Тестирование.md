# Тестирование
---
### 1. Какая операционная система НЕ рекомендуется для установки ISPmanager?
- [ ] A) AlmaLinux 9  
- [ ] B) Debian 12  
- [ ] C) Ubuntu с предустановленным Apache  
- [ ] D) CentOS 7

<details>
<summary>Показать ответ</summary>

**C** — ISPmanager требует **чистую систему без предустановленного веб-сервера, MySQL или других конфликтующих пакетов**. Ubuntu с Apache вызовет ошибки при установке.
</details>
---
### 2. Где в ISPmanager настраиваются автоматические резервные копии?
- [ ] A) В разделе «Сайты»  
- [ ] B) В разделе «Пользователи»  
- [ ] C) В разделе «Резервные копии → Планировщик»  
- [ ] D) В настройках SSH

<details>
<summary>Показать ответ</summary>

**C** — автоматические бэкапы настраиваются в разделе **«Резервные копии»**, подраздел **«Планировщик»**. По умолчанию они отключены.
</details>
---
### 3. Какой протокол обеспечивает шифрование при передаче бэкапов из ISPmanager?
- [ ] A) FTP  
- [ ] B) HTTP  
- [ ] C) SFTP  
- [ ] D) SMTP

<details>
<summary>Показать ответ</summary>

**C** — **SFTP** (SSH File Transfer Protocol) шифрует весь трафик, включая аутентификацию и данные. FTP и HTTP передают данные в открытом виде.
</details>
---
### 4. Почему ISPmanager не подходит для микросервисной архитектуры?
- [ ] A) Он не поддерживает Linux  
- [ ] B) Он работает только с WordPress  
- [ ] C) Он ориентирован на монолитные сайты и не поддерживает IaC, CI/CD, Kubernetes  
- [ ] D) Он слишком дорогой

<details>
<summary>Показать ответ</summary>

**C** — ISPmanager создан для управления **традиционными веб-сайтами, почтой и БД**, но не предоставляет инструментов для оркестрации контейнеров, автоматизированного развёртывания или наблюдаемости микросервисов.
</details>

### 5. Что делает Let’s Encrypt в ISPmanager?
- [ ] A) Шифрует бэкапы  
- [ ] B) Автоматически выдаёт бесплатные SSL-сертификаты для доменов  
- [ ] C) Блокирует DDoS-атаки  
- [ ] D) Устанавливает обновления ОС

<details>
<summary>Показать ответ</summary>

**B** — Let’s Encrypt интегрирован в ISPmanager и позволяет **автоматически получать и обновлять SSL-сертификаты** для всех добавленных доменов — бесплатно и без ручного вмешательства.
</details>
