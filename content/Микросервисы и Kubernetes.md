## [[m9t1_Архитектура микросервисов]]
Микросервисы — это архитектурный стиль, при котором приложение состоит из **маленьких, независимых сервисов**, общающихся по сети. 
DevOps-инженер играет ключевую роль в их развёртывании, наблюдаемости и отказоустойчивости.
## [[m9t2_Что такое микросервисы]]
- Преимущества vs монолит
- Коммуникация между сервисами → API Gateway, Service Mesh

## [[m9t3_Kubernetes Administration]]
- Компоненты: control plane, worker nodes
- kubectl → deployments, services, configmaps, secrets
- Helm → управление приложениями
- Troubleshooting → логи, describe, exec
