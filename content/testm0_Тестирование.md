# Тест по модулю 0
---
### 1. Что является главной целью DevOps?
- [ ] A) Заменить ручное тестирование автоматизированным  
- [ ] B) Ускорить доставку ПО без потери качества и надёжности  
- [ ] C) Внедрить Docker и Kubernetes во все проекты  
- [ ] D) Устранить необходимость в системных администраторах

<details>
<summary>Показать ответ</summary>

**B** — DevOps фокусируется на балансе скорости и надёжности, а не на инструментах.
</details>
---
### 2. Какой из этапов НЕ входит в замкнутый SDLC в DevOps?
- [ ] A) Planning  
- [ ] B) Development  
- [ ] C) Deployment  
- [ ] D) Budgeting

<details>
<summary>Показать ответ</summary>

**D** — Budgeting не является частью жизненного цикла разработки ПО.
</details>
---
### 3. Чем Continuous Deployment отличается от Continuous Delivery?
- [ ] A) Использует другие инструменты (например, Argo CD вместо Jenkins)  
- [ ] B) Требует ручного подтверждения перед деплоем в production  
- [ ] C) Автоматически деплоит каждый успешный коммит в production  
- [ ] D) Применяется только в облаке

<details>
<summary>Показать ответ</summary>

**C** — Continuous Deployment = автоматический деплой; Delivery = готовность, но ручной запуск.
</details>
---
### 4. Команда использует Scrum, GitLab CI и мониторинг через Prometheus с SLO. Какие подходы они применяют?
- [ ] A) Только DevOps  
- [ ] B) Agile + DevOps + SRE  
- [ ] C) Только SRE  
- [ ] D) DevOps + Waterfall

<details>
<summary>Показать ответ</summary>

**B** — Scrum (Agile), GitLab CI (DevOps), SLO (SRE).
</details>

### 5. Что такое «error budget» в SRE?
- [ ] A) Бюджет на исправление багов  
- [ ] B) Допустимое количество сбоев за период  
- [ ] C) Сумма штрафов за downtime  
- [ ] D) Время, выделенное на отладку

<details>
<summary>Показать ответ</summary>

**B** — Error budget — ключевая метрика надёжности в SRE.
</details>
