# Тестирование

Заполни чекбоксы, как будто отвечаешь. Потом проверь себя в сворачиваемых блоках.

### 1. Какой командой можно посмотреть, какие процессы используют порт 80?
- [ ] A) `netstat -p 80`  
- [ ] B) `lsof -i :80`  
- [ ] C) `ss -tuln | grep 80`  
- [ ] D) `ps aux | grep 80`

<details>
<summary>Показать ответ</summary>

**B** — `lsof -i :80` показывает процессы, слушающие порт 80.  
(Также верно: `ss -tulnp | grep :80`, но в вариантах он не указан полностью.)
</details>

### 2. Что делает команда `grep "ERROR" /var/log/app.log | wc -l`?
- [ ] A) Удаляет все строки с "ERROR"  
- [ ] B) Считает количество строк, содержащих "ERROR"  
- [ ] C) Заменяет "ERROR" на пустую строку  
- [ ] D) Выводит первые 10 строк с "ERROR"

<details>
<summary>Показать ответ</summary>

**B** — `wc -l` считает строки, прошедшие через `grep`.
</details>

### 3. Как правильно обработать ошибку в Bash-скрипте?
- [ ] A) Игнорировать — скрипт сам остановится  
- [ ] B) Использовать `command || echo "Ошибка"`  
- [ ] C) Использовать `set -e` в начале скрипта  
- [ ] D) Перезапустить сервер

<details>
<summary>Показать ответ</summary>

**C** — `set -e` заставляет скрипт завершиться при любой ошибке. Это стандартная практика.
</details>

### 4. Какой файл нужно редактировать, чтобы добавить задачу в cron для текущего пользователя?
- [ ] A) `/etc/crontab`  
- [ ] B) `crontab -e`  
- [ ] C) `/var/spool/cron/crontabs/root`  
- [ ] D) `systemctl edit cron`

<details>
<summary>Показать ответ</summary>

**B** — `crontab -e` открывает редактор для cron-задач текущего пользователя. Безопасно и удобно.
</details>

### 5. Почему важно заключать переменные в кавычки: `"$file"` вместо `$file`?
- [ ] A) Так красивее  
- [ ] B) Чтобы ускорить выполнение  
- [ ] C) Чтобы избежать ошибок при пробелах в имени файла  
- [ ] D) Чтобы скрыть значение переменной
<details>
<summary>Показать ответ</summary>

**C** — если `$file` содержит пробелы (например, `"my file.txt"`), то `rm $file` превратится в `rm my file.txt` → ошибка. Кавычки сохраняют целостность значения.
</details>
